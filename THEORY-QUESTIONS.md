## Study Strategy

### How to Study These Questions

1. **Read the answer** on BFE.dev (full explanation, not just title)
2. **Write 1-sentence summary** in your own words
3. **Link related coding problem** if one exists (e.g., Q#4 "Closure" → Problem #14 "memo")
4. **Explain to someone** (rubber duck debugging or speak aloud)
5. **Mark when done** and add revision date

### Recommended Schedule

**Week 1-2 (Now):**
- Phase 1: Read 5 questions: #5, #3, #4, #6, #18 (one per day, ~10 min each)
- Code while learning: Start coding problems related to closures, promises

**Week 3-4:**
- Phase 1: Finish 11 remaining questions
- Phase 2: Start 6 browser questions

**Week 5-6:**
- Phase 3: All 6 network questions
- Phase 4: All 4 design pattern questions

**Week 7+:**
- Phase 5: Security basics
- Phase 6-7: Advanced topics when time permits

### Tips

- ✓ Study questions **in order** (dependencies exist: var/let/const → this → closures → promises)
- ✓ Study **1-2 per day** (not all at once)
- ✓ Link to related **coding problems** as you solve them
- ✓ Use spaced repetition: revisit hard questions weekly
- ○ DON'T memorize — understand the concept
- ○ DON'T skip Phase 1 & 3 (most asked in interviews)
- ○ DON'T study all 173 questions (diminishing ROI after 40)
